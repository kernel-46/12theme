Pratyaya is a real-time voice-to-voice assistive layer designed for Karnataka's 1092 helpline, acting as an AI co-pilot that sits between the citizen and the agent to ensure every issue is accurately heard, interpreted, and verified before any response is committed. The name itself is Sanskrit for conviction and verified understanding — a fitting description of the system's core purpose. Pratyaya never replaces the human agent, never auto-resolves a call, and never acts without confirmation. Its sole mission is to guarantee that the agent and the citizen mean exactly the same thing before any action is taken.
The problem Pratyaya addresses is both practical and urgent. Karnataka's 1092 helpline receives calls in Kannada, Hindi, English, and a wide range of dialect-rich variants including Dharwad Kannada, Mangaluru Kannada, Hyderabad-Karnataka Kannada, and code-mixed Kanglish. Citizens calling this helpline are rarely calm — they are reporting harassment, distress, emergencies, or unresolved grievances. Agents are skilled but often comfortable in only one language, and in a high-pressure call environment, a misheard word or a missed dialect cue can result in a wrong service request, a delayed dispatch, or a citizen who hangs up feeling unheard. The cost of misunderstanding here is not inconvenience it is harm.
The system is built on five tightly integrated components. First, a multilingual streaming ASR layer using AI4Bharat's IndicConformer and BharatGen's Shrutam2 produces word-level transcripts with per-word confidence scores, handling native code-switching between Kannada-English and Hindi-English. Second, a dialect classifier built on a wav2vec2-based model trained on IndicVoices data identifies whether the caller is speaking Bangalore-standard, Dharwad, or Mangaluru Kannada, and routes audio to the most appropriate model variant. Third, an intent and entity extraction layer powered by Sarvam-1, a 2-billion-parameter India-native language model running entirely within Indian jurisdiction, converts the transcript into a structured representation covering issue type, location, named persons, urgency markers, and factual claims. Fourth, a verification loop paraphrases the extracted issue back to the citizen in their own dialect and asks a single closed-form confirmation question, capturing verbal yes/no responses including dialect-specific affirmations like "haudu" or "haan." Fifth, a live agent dashboard displays the running transcript, structured interpretation, dialect tag, sentiment trajectory, and a confidence meter — all editable and reversible — so the agent always knows exactly what the AI heard and what it is about to commit.
Beyond this foundation, Pratyaya introduces four key innovations. The Confidence-Gated Verification Loop runs IndicConformer and Shrutam2 in parallel and treats disagreement between their outputs as a formal signal. Rather than silently averaging outputs, the system operates as a three-state machine VERIFIED, CLARIFY, or HANDOVER and escalates appropriately when confidence drops. The Multimodal Sentiment Fusion layer captures prosodic signals like pitch contour, speaking rate, jitter, and breath patterns alongside lexical sentiment, producing a six-dimensional emotional state — distress, urgency, anger, fear, confusion, calm updated continuously through the call. This means the system can detect a citizen in crisis from their voice alone, independent of what words they use.
The Graceful Handover Protocol ensures that when the system needs to transfer to a human agent, the transition is warm and seamless. A bridging line is spoken in the citizen's own dialect, the agent dashboard is pre-populated with full context, and the citizen never has to repeat themselves. Every handover is logged in a hash-chained audit ledger. Finally, the Continuous Learning mechanism treats every confirmed verification as a labelled training pair and every correction as an even more valuable error signal, allowing the system to improve over time on the exact dialects and distress registers it actually encounters.
On privacy, Pratyaya applies PII redaction at the call edge before any data is stored or passed to language models. All inference runs on locally hosted open-source models within Indian jurisdiction no audio leaves the country, no foreign API is involved, and every access to archived calls is itself logged.
The rollout follows a shadow-mode-first approach where the system runs on live calls without speaking, until verification accuracy crosses a defined threshold per dialect. The handover path is live from day one as the foundational safety net. Deployment is entirely on open-source, Indian-cloud-compatible infrastructure AI4Bharat, BharatGen, Sarvam, and Wav2Vec2 models orchestrated via FastAPI, Kafka, PostgreSQL, and Kubernetes on MeghRaj or State Data Centre  with no closed-source critical path and no foreign dependency.
Over time, Pratyaya evolves from a voice verification tool into a real-time civic sensor  a continuous map of what citizens are calling about, from which districts, in which dialects, and with what emotional urgency. The same loop that prevents misunderstanding produces, as a natural by-product, one of the most valuable Indian-language conversational datasets any State government will ever own.
Pratyaya — the citizen is heard, the understanding is verified, and the agent always knows what the AI thinks before the AI ever speaks.
